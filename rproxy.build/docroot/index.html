About
This server is a managed instance of the Next-Generation Clustered Heat Map (NG-CHM) system developed at MD Anderson Cancer Center. A brief description of the server architecture is available here.
Viewing NG-CHMs
The NG-CHM Viewer interface can be used to browse and view NG-CHMs after they have been uploaded to the server.
Managing NG-CHMs
If accessible, the NG-CHM Manager interface can be used to upload NG-CHMs to the system, delete NG-CHMs, etc.
Using this server with R/RStudio
To use this server with the NG-CHM R library, use the chmCreateServer function. Replace 'name' with your name for this server, and replace 'viewer.url' with the URL to the viewer interface. 
> library(NGCHM)
> chmCreateServer ('name', 'viewer.url')
Alternatively, you can specify the server details in an initialization file as detailed in the library documentation.
Creating an example NG-CHM
The following commands will create a simple NG-CHM and upload it to server 'name'. The server parameter can be omitted if it is the default (or only) server. 
> data(TCGA.GBM.EXPR)
> chm = chmNew ('example', TCGA.GBM.EXPR[1:50,1:50])
> chmMake (chm, server='name')
> chmInstall (chm, server='name')
The following command will open the new NG-CHM in a new browser window: 
> browseURL(chmGetURL (chm, server='name'))
In the RStudio environment, the following command will display the new NG-CHM in the RStudio viewer pane: 
> getOption('viewer')(chmGetURL (chm, server='name'))
