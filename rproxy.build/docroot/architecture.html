Server Architecture
This page describes the internal architecture of this server. It is not necessary to know or understand this material in order to simply use the system.

This server consists of six Docker containers in total.

There are two data containers:

  • a data container containing the NG-CHMs uploaded to the system, and
  • a data container containing a database of read-only linkout definitions.
There are three 'backend' server containers:
  • an NG-CHM viewer container,
  • an NG-CHM manager container,
  • a DyCE container for creating content-specific linkouts.
The three backend server containers are connected to a private (internal) Docker network.

The final container implements a reverse-proxy server with a published interface on a public network. The reverse proxy container is also connected to the private network and forwards client requests to the appropriate 'backend' container.
